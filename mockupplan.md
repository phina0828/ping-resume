視覺設計師作品集：核心功能與交互規範計劃書
1. 專案概述
本專案目標是生成一個具備「高質感 Mockup 互動」與「雲端管理後台」的平面設計師作品集。網站需能針對不同作品類別，自動切換最合適的展示模式。
2. 技術規格
前端框架：React
樣式工具：Tailwind CSS
圖示庫：Lucide-React
雲端整合：Firebase (Auth, Firestore)
視覺風格：極簡、高級留白、標誌性紅色 (#DC2626) 點綴。
3. 四大核心展示功能需求
A. 3D 翻書動畫 (3D Flip Book Mockup)
適用類別：排版設計、企業專案、書籍裝幀。
視覺設計：
模擬實體雜誌，具備左右對開效果。
書脊處需有漸層陰影 (gradient-to-l/r from-black/10)。
交互邏輯：
點擊「下一頁」時，單頁頁面以書脊為軸心進行 180 度翻轉 (rotateY(-180deg))。
使用 CSS preserve-3d 與 backface-hidden 確保翻轉過程正反面內容正確切換。
每翻一次顯示 2 張新圖片。
B. IG 貼文輪播 (Instagram Style Carousel)
適用類別：品牌設計、社群視覺、圖文編排。
視覺設計：
完整模擬 Instagram UI（包含頭像、名稱、地點、心形、留言、收藏圖示）。
圖片區域為 1:1 正方形 (aspect-square)或3:4直式。
交互邏輯：
橫向輪播功能。
下方配備紅色圓點分頁指示器，當前頁面需有縮放與加深效果。
C. PDF 滾動預覽 (PDF Interactive Reader)
適用類別：企業報告、ESG 報告書、品牌手冊。
功能設計：
嵌入 PDF 瀏覽器（使用 iframe）。
必須支援原生滾動，使用者能直接在預覽窗格中上下滑動。
同步提供「下載 PDF」功能按鍵與「外部開啟連結」。
視覺提示：加入「互動模式」標籤與呼吸燈效果，提示訪客此區域可滾動。
D. 純淨並排展示 (Pure Simple Stack)
適用類別：平面設計、海報系列、UI 頁面流程。
視覺設計：
移除所有 Mockup 裝飾，還原原始設計稿比例。
圖片採垂直大圖並排 (Simple Stack)。
圖片間距加大，輔以細微的懸浮放大效果 (hover:scale-[1.01])。
4. 管理控制台需求 (Visual CMS)
A. 圖像化管理介面
圖樣預覽：輸入圖片網址後，管理面板需立刻顯示封面與內頁縮圖。
動態多圖管理：
提供「新增一張圖」按鈕，每點擊一次增加一個獨立輸入框。
每個圖片欄位旁需有「刪除」按鈕。
類別分流器：提供下拉選單切換「品牌設計(IG)」、「排版設計(翻書)」、「平面設計(並排)」、「PDF 預覽」。
B. 軟體工具編輯器
動態清單：可自行新增工具名稱（如 Cursor, Photoshop）與圖示路徑。
圖示對應：前台需自動渲染管理面板中設定的所有工具圖示。
5. 安全與穩定性規範 (React 實作守則)
防崩潰機制：所有雲端獲取的資料必須透過安全轉換函式（如 safeStr），確保不會意外渲染 React 物件，避免 "Objects are not valid as a React child" 錯誤。
Firebase 路徑：資料庫路徑必須為符合權限規範的 6 段式偶數路徑（例如：/artifacts/{appId}/public/data/collection/document）。
封閉環境適配：程式碼必須封裝於單一檔案，並使用 Lucide 圖示或內嵌 SVG，不依賴外部未知資產。
